API Documentation

Main.PowerMonitor.CPUPowerMonitor — Type

Structure to monitor CPU power consumption using RAPL (Running Average Power Limit).

Fields: - prevenergy::Float64: Energy consumption at the previous measurement (in joules). - prevtime::Float64: Time of the previous measurement (in seconds). - initialized::Bool: Whether the monitor has been initialized.

Constructor: CPUPowerMonitor(): Initializes a new monitor with default values.

source
Main.PowerMonitor.init_gpu_monitor — Method

Initializes GPU power monitoring by ensuring CUDA and NVML are available.

Throws an error if CUDA or NVML support is not detected.

source
Main.PowerMonitor.log_power_data — Method

Logs power data to a CSV file.

Parameters: - filename::String: Name of the CSV file to save the data. - powerdata::PowerData: The power data to log.

The CSV file contains three columns: Time(s), GPUPower(W), and CPUPower(W).

source
Main.PowerMonitor.monitor_power — Function

Monitors power consumption of the GPU and CPU over a specified duration.

Parameters: - duration::Float64: Total duration of monitoring (in seconds). - interval::Float64: Sampling interval for power measurements (in seconds).

Returns: - PowerData: A structure containing time, GPU power, and CPU power data.

source
Main.PowerMonitor.read_cpu_power — Method

Reads CPU power consumption using the Linux sysfs interface.

Parameters: - cpu_monitor::CPUPowerMonitor: The CPU power monitor state.

Returns: - Float64: The calculated CPU power consumption (in watts).

Throws: - Error: If unable to read from the RAPL sysfs interface.

source
Main.PowerMonitor.terminate_gpu_monitor — Method

Placeholder function for GPU power monitoring cleanup. Currently, no explicit cleanup is required.

Monitors power consumption (GPU and CPU) during the execution of a code block.
